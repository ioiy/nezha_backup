🛡️ 哪吒面板 V2 自动备份与管理工具 (Nezha Backup)

专为哪吒探针 (Nezha Panel) V2 打造的一站式、全自动、带精美交互面板的 S3 数据备份与灾备恢复脚本。即使是 512MB 内存的小鸡也能稳定运行，彻底告别坏档与数据丢失风险！

✨ 核心亮点 (Features)

🖥️ 精美交互式 UI：告别繁琐的命令行参数，全部操作均可通过纯中文的可视化菜单完成。

☁️ S3 协议全兼容：基于 Rclone 强力驱动，完美兼容各类云厂商的 S3 存储桶（内置 no_check_bucket 机制，支持严格权限的 AK/SK 密钥）。

🪶 512M 低配机福音：专为小内存机器优化。手动运行时采用极低内存参数；定时运行时自动挂载 nice 和 ionice 最低优先级，完全不卡面板！

🩺 双重预检机制 (防毁档)：备份前自动执行 SQLite 数据库健康扫描与可用磁盘空间比对，发现异常立即终止并报警，绝不把损坏或不完整的数据传到云端覆盖！

🔒 AES-256 军事级加密：内存级流式加密，你的探针数据（包含极度敏感的 GitHub Token、服务器密码）在公有云上也是绝对安全的密文。

🗂️ 长效白名单策略 (GFS)：不仅仅是“保留最近 14 天”，开启 GFS 策略后，脚本会永远保留每个月 1 号的备份文件作为基准存档。

🚑 一键可视化定点恢复：独创的 S3 备份管理中心！可以直接在面板里列出云端备份，输入序号即可一键全自动下载、解密并覆盖重启面板。

🤖 全能 Telegram 播报：成功或失败均可推送到 TG，自带详细日志及当前服务器的「剩余磁盘、内存」监控播报。

🚀 极速安装与使用 (Quick Start)

只需在安装了哪吒面板的服务器上，使用 root 用户运行以下一键命令：

curl -O [https://raw.githubusercontent.com/ioiy/nezha_backup/main/bf.sh](https://raw.githubusercontent.com/ioiy/nezha_backup/main/bf.sh) && chmod +x bf.sh && ./bf.sh


运行后将进入主菜单，您可以按照以下顺序进行初次配置：

选择 2 配置您的 S3 存储桶信息。

选择 4 配置 Telegram Bot 通知（可选）。

选择 9 设置 AES-256 备份加密密码（可选，极度推荐）。

选择 1 立即执行一次手动备份测试，下方将显示实时上传进度条！

选择 6 开启自动定时备份（默认每天凌晨 3:00 静默执行）。

💻 进阶：如何在 PC 本地解密备份文件？

如果您开启了备份加密（文件后缀为 .tar.gz.enc），且希望在自己的 Windows 或 Mac 电脑上直接查看备份内容，请使用以下命令：

前置要求：电脑上已安装 OpenSSL（Windows 用户可以直接使用安装 Git 时自带的 Git Bash 终端）。

执行以下解密命令：

openssl enc -d -aes-256-cbc -pbkdf2 -salt -in 您的加密文件.tar.gz.enc -out 解密后的文件.tar.gz -k "您的加密密码"


完成后，您将得到一个标准的 .tar.gz 压缩包，使用 WinRAR 或 7-Zip 即可解压并查看里面的 sqlite.db 和配置数据。

🔄 脚本更新

本脚本内置了强效的平滑升级功能。若想获取最新功能，只需在主菜单中输入 7，脚本便会自动绕过 CDN 缓存获取最新的 GitHub 版本并热重载，无需重新配置任何参数！

⚠️ 注意事项

本脚本默认备份 /opt/nezha 目录，请确保您的哪吒面板安装在默认路径。

恢复备份时会强制覆盖当前的 /opt/nezha 目录并自动重启 nezha-dashboard 服务，请谨慎操作。

强烈建议开启 TG 失败通知，以便在服务器发生磁盘爆满、数据库损坏或 S3 欠费时第一时间接管！
